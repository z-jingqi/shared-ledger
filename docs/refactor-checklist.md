# Refactor Checklist

本文档用于跟踪按新设计重构时必须清理的旧功能、假入口和遗留资源。它不是产品设计源文档；产品规则以 `docs/product-design.md` 为准。

## Phase 1

- 登录 / 注册
  - 状态：已完成
  - 要求：用户名密码登录；用户名、密码、确认密码注册；密码 eye 独立控制；登录注册状态不共享。

- App Shell / 首页 / 账本上下文
  - 状态：已完成
  - 要求：底部导航改为首页、记录、中间添加、分析、我的；账本为顶部上下文；有账本进入上次账本，没有账本进入创建账本。

- 记录模块
  - 状态：已完成
  - 要求：搜索筛选真实可用；隐藏标签、账户、成员；分类必填；明细真实回填；保存真实调用 API。

- AI 助手
  - 状态：已完成
  - 要求：全屏聊天工作台；无语音；用户消息气泡、AI 无背景；附件在统一 composer 内预览；确认保存后才处理附件。

## Phase 2

- 设置 / 我的
  - 状态：已完成
  - 已移除深色模式、隐私设置、通知设置、标签管理和账号页假入口。
  - 数据导出保留，真实调用导出 API。

- 账本设置
  - 状态：部分完成
  - 已移除预算设置。
  - 默认货币创建后只读。
  - 删除账本只有创建者可见。
  - 成员退出账本需要真实 API 支持；没有 API 时不能显示假按钮。
  - 管理员可编辑成员数据开关默认关闭；没有后端支持时不显示或标记为后续任务，不做假开关。

- 成员 / 邀请
  - 状态：部分完成
  - 邀请必须检查已是成员、已有待处理邀请、权限不足等状态。
  - 链接邀请已移除。
  - 成员角色修改必须真实调用 API。
  - 移除成员、退出账本等高风险操作必须确认。

- 文件处理 / 导入
  - 状态：部分完成
  - 已去掉独立导入主页面作为一级入口，旧路由改为兼容重定向。
  - 文件上传已支持从中间添加按钮和 AI 附件进入。
  - 处理任务必须进入全局处理状态。
  - 完成后进入待确认记录。
  - 独立导入页代码已删除，仅保留待确认记录和导入历史页。

## Acceptance

- 每个页面都要和对应设计图比对。
- 不允许保留不可点击、不可用、只展示状态的假功能。
- 任何按钮或链接如果显示在 UI 中，必须能完成真实操作或进入真实页面。
- 旧代码和旧资源在对应功能迁移完成后必须删除。
- 每个阶段至少运行：
  - `pnpm --filter @shared-ledger/web typecheck`
  - `pnpm --filter @shared-ledger/web lint`
  - `pnpm --filter @shared-ledger/web test`
