# Page Design Inventory

本文档定义全新设计阶段需要产出的页面设计图和状态图。它不基于当前实现或旧设计稿，只用于指导后续 UI 设计、前端实现和验收。

## 1. 设计图产出原则

每个页面不只需要默认态，还需要覆盖真实使用中的关键状态：

- 空状态
- 有数据状态
- 加载状态
- 错误状态
- 权限不足状态
- 弹窗打开状态
- 表单校验状态
- 底部固定按钮状态
- 全局文件处理状态
- AI 助手打开状态

移动端优先，所有设计图默认按照手机视口设计。

建议每张图命名格式：

```text
模块序号-页面序号-状态名称.png
```

示例：

```text
01-home-01-default.png
01-home-02-book-switcher-open.png
```

## 2. 全局设计图

### 2.1 App Shell

需要设计：

1. 主页面基础框架
   - 固定顶部区域
   - 可滚动内容区
   - 全局处理状态条
   - AI 悬浮入口
   - 底部导航
   - 中间圆形添加按钮

2. 子页面基础框架
   - 固定标题栏
   - 可滚动内容区
   - 固定底部主按钮
   - 无底部导航

3. 登录前基础框架
   - 不显示底部导航
   - 不显示全局处理状态
   - 不显示 AI 入口

### 2.2 底部导航与添加按钮

需要设计：

1. 底部导航默认态
   - 首页 / 记录 / 添加 / 分析 / 我的
   - 中间 `+` 圆形按钮嵌入导航栏
   - 导航栏中间圆弧 gap

2. 底部导航选中态
   - 首页选中
   - 记录选中
   - 分析选中
   - 我的选中

3. 添加按钮打开态
   - `+` 旋转成 `x`
   - 背景遮罩
   - 底部操作面板
   - 操作项：手动记录、上传文件

4. 添加按钮打开后点击遮罩关闭态

5. 上传文件触发系统选择前的状态
   - 面板关闭
   - 文件选择器由系统接管，无需设计系统弹窗

## 3. 登录与初始化

### 3.1 登录页

需要设计：

1. 默认态
   - 用户名
   - 密码
   - 密码 eye icon 默认关闭
   - 登录按钮
   - 注册入口

2. 密码可见态
   - eye icon 打开
   - 密码明文显示

3. 表单错误态
   - 用户名为空
   - 密码为空
   - 密码错误
   - 网络错误

4. 提交中状态
   - 登录按钮 loading
   - 输入不可重复提交

### 3.2 注册页

需要设计：

1. 默认态
   - 用户名
   - 密码
   - 确认密码
   - 两个密码字段都有 eye icon
   - 注册按钮
   - 登录入口

2. 表单错误态
   - 用户名为空
   - 密码少于 6 位
   - 两次密码不一致
   - 用户名已存在

3. 提交中状态

### 3.3 应用初始化

需要设计：

1. 获取账本 loading

2. 没有账本
   - 直接进入创建账本页

3. 有账本
   - 进入上一次使用账本首页

4. 上次使用账本不可用
   - 进入第一个可用账本

### 11.2 AI 聊天面板

需要设计：

1. 默认打开态
   - 当前账本上下文
   - 消息列表
   - 输入框
   - 附件按钮
   - 发送按钮
   - 无语音按钮
   - 消息不显示时间
   - 用户消息使用有背景色气泡
   - AI 消息不使用背景气泡
   - 消息最大宽度为 100%，保留左右页面间距

2. 空会话状态
   - 推荐问题

3. 用户消息发送中

4. AI 回复中

5. AI 回复失败

6. 长消息状态

7. 消息内链接按钮状态

8. 消息内确认按钮状态

9. 附件菜单打开状态
   - 拍照
   - 从相册选择
   - 选择文件

10. 附件已选择状态
   - 发送前在输入框整体容器内部上方显示附件预览

   - PDF、Excel、CSV 等文件显示同尺寸文件卡片
   - 文件卡片显示扩展名和文件名
   - 可移除附件
   - 多附件横向排列，超出后横向滚动
   - 输入文字后，附件预览保持在上方，文本在下方输入行

11. 多行输入状态
   - 输入框高度增长
   - 达到最大高度后内部滚动
